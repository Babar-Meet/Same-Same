## Purpose

Unicode contains many characters from different scripts (Latin, Greek, Cyrillic, etc.) that look identical or very similar to one another. These are known as **homoglyphs** or **lookalikes**. Attackers exploit these in **homograph attacks** — registering domain names or crafting filenames that visually mimic legitimate ones but decode to different Unicode code points.

This project provides a searchable, filterable reference to:

- Identify which characters look like which
- See Unicode names, code points, and similarity ratings
- Copy lookalike characters for testing, input methods, or security analysis
- Differentiate between strict visual lookalikes and related-but-not-identical alternatives

It is intended for security researchers, penetration testers, typographers, linguists, and developers working with Unicode normalization or input validation.

## Features

- **Complete a–z / A–Z coverage** — every Latin letter mapped to its lookalikes across Greek, Cyrillic, and mathematical scripts.
- **Digit lookalikes (0–9)** — digits mapped to visually similar letters and mathematical styles.
- **Symbol lookalikes** — spaces, dashes, brackets, punctuation, mathematical operators, arrows, currency signs, and more.
- **Mathematical style variants** — Bold, Bold Italic, Sans-Serif, Sans-Serif Bold, and Monospace for every letter and digit.
- **Fullwidth variants** — CJK-compatible fullwidth equivalents for every letter, digit, and many symbols.
- **Similarity rating** — each entry is labelled `Exact`, `High`, `Medium`, or `Low`.
- **Unicode code point display** — every entry shows its `U+XXXX` code.
- **Alt code display** — Windows Alt-code input method shown for characters in the 0–255 range.
- **Common accented-character Alt codes** — reference table for accented Latin variants (À, É, ñ, etc.).
- **Per-character copy** — click a button to copy any single character to the clipboard.
- **Copy All** — copy every character in a section at once.
- **Full-text search** — filter sections by original character, Unicode name, code point, Alt code, or description.
- **Single-character quick filter** — typing exactly one character shows only its section.
- **Exact-key mode** — typing an exact key shows only that section.
- **Whitespace shortcut** — typing a space character filters to the space section.
- **Related symbols section** — non-lookalike but thematically related alternatives (e.g., other currency signs) shown separately.
- **Keyboard shortcut** — `/` focuses the search input from anywhere on the page.
- **No external dependencies** — single HTML file, zero framework, zero libraries.

### Page Load Sequence

1. The page loads and executes the inline `<script>` at the end of `<body>`.
2. `createTables()` (line 1252) is called, which:
   - Clears the `tables` container.
   - Calls `appendSections()` with the main `data` object and then with `relatedSymbolData`.
3. `appendSections()` iterates over each key in the data object. For each key:
   1. Builds a search index string via `buildSectionSearchIndex()`.
   2. Creates a `<section>` element with a `<h2>` header, a "Copy All Characters" button, and an HTML `<table>`.
   3. Each table row contains: character (monospace), Copy button, similarity rating, Unicode name, Alt code, `U+XXXX` code, and a description string.
4. Event listeners are attached:
   - `searchInput` `input` event for filtering.
   - `document` `keydown` event for `/` search focus.
   - Per-row Copy button `onclick` handlers.
   - Per-section "Copy All" button `onclick` handler.

### Search/Filter Mechanics

- Each `<section>` stores a pre-computed, flattened search string in `dataset.search` (built by `buildSectionSearchIndex`).
- On every keystroke in the search input, the listener computes a set of lowercased tokens and tests each section's search index against all tokens.
- Special handling for single-character input (shows only that specific section), exact key match, and whitespace detection.

### Copy Mechanism

- Uses the **Clipboard API** (`navigator.clipboard.writeText`).
- Single-character copy: writes the character in the current row.
- "Copy All Characters": joins all characters in the section (including the original) and writes the concatenated string.

## Core Components

### `makeEntry(char, code, name, sim, allowed)`

Factory function (line 73) that returns a uniform entry object:
- `char` — the Unicode character
- `code` — 4-digit uppercase hex code point (e.g. `"03B1"`)
- `name` — Unicode character name
- `sim` — similarity: `"Exact"`, `"High"`, `"Medium"`, `"Low"`
- `allowed` — whether the character is generally permitted in identifiers

### `toHex(value)`

Converts a numeric code point to a zero-padded 4-digit uppercase hex string (line 81).

### `dedupeEntries(entries)`

Filters an array of entry objects to remove duplicates based on `char` + `code` (line 84). Used when merging hand-curated extras with mathematically generated variants.

### `formatAltCode(hexCode)`

Computes a Windows Alt-code string from a hex Unicode code point (line 94). For code points ≤ 255, it formats as `Alt + NNNN (NNN)` using the decimal value. For larger code points, it gives `Alt + NNNNN`.

### `getOrigDisplay(orig)`

Returns `"[space]"` for the space character; otherwise returns the character itself (line 103). Used in headings and descriptions.

### `getOriginType(unicodeName)`

Classifies a Unicode name into a category (line 105): `"Fullwidth compatibility"`, `"Greek script"`, `"Cyrillic script"`, `"Mathematical style"`, `"Roman numeral"`, `"Whitespace"`, or `"Unicode"`.

### `buildCharDescription(orig, item, rowIndex, isRelated)`

Generates a human-readable description for each table row (line 116). For the first row (the original character), it labels it as the base reference. For subsequent rows, it describes the script origin and how it relates to the base character.

### `getCommonAltCodesText(orig)`

Returns an Alt-code reference string for accented variants of a given letter (line 708). Uses the static `commonAltCodeSets` lookup table.

### `buildSectionSearchIndex(orig, sectionRows, commonAltText, isRelated)`

Pre-computes a single searchable string for a section (line 1109), combining the original character, its display form, its lowercase form, its code point, common Alt codes, the section type, and all sub-entry data. Stored in `dataset.search`.

### `appendSections(container, sourceData, titlePrefix, isRelated)`

The main DOM generation function (line 1145). Creates sections, headers, copy buttons, and tables for each key in the source data object. Accepts a flag to render related-symbol sections differently.

## How It Works

1. Open `index.html` in a browser.
2. All character sections are rendered immediately as static HTML tables — no network requests, no loading state.
3. Each section is headed by the original character (e.g., `a`, `5`, `-`) and lists all known lookalikes.
4. Use the **filter box** at the top to narrow down sections. The filter searches across original characters, Unicode names, code points, Alt codes, and descriptions.
5. When you find a character you need, click its **Copy** button. The character is placed on your system clipboard via the Clipboard API.
6. To copy every character in a section at once, click **Copy All Characters**.
7. Press `/` at any time to focus the search input.
8. Accented-letter sections (e.g., `a`, `e`, `n`, `o`) also display a common Alt-code reference table for Windows users.

From a code perspective:

- On page load, `createTables()` iterates over a large static JavaScript object (`data`) that maps each original character to an array of lookalike entries.
- Each entry has a `char`, `code` (hex), `name`, `sim` (similarity), and `allowed` flag.
- Entries come from two sources: **hand-curated** mappings (Greek/Cyrillic cross-script lookalikes, Roman numerals, Cherokee, etc.) and **programmatically generated** mappings (Fullwidth variants, Mathematical Alphanumeric Symbol block styles).
- For letters A–Z and a–z, the script generates Fullwidth (U+FF21–U+FF5A), Mathematical Bold (U+1D400–U+1D433 / U+1D41A–U+1D44D), Bold Italic, Sans-Serif, Sans-Serif Bold, and Monospace variants on the fly using `String.fromCodePoint()`.
- For digits 0–9, it generates Fullwidth and the same mathematical set including Double-Struck.
- For symbols, all entries are hand-curated in `symbolData`.
- The `appendSections` function creates all DOM elements. Each section's `dataset.search` stores a pre-computed search string for fast client-side filtering.
- Filtering is O(n) over sections, testing every token against the pre-computed index. Single-character and exact-key modes short-circuit for instant response.
